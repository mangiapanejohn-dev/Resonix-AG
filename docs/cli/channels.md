# `resonix channels`

Manage chat channel accounts and their runtime status on the Gateway.

Related docs:

- Channel guides: [Channels](/channels/index)
- Gateway configuration: [Configuration](/gateway/configuration)

## Common commands

```bash
resonix channels list
resonix channels status
resonix channels capabilities
resonix channels capabilities --channel discord --target channel:123
resonix channels resolve --channel slack "#general" "@jane"
resonix channels logs --channel all
```

## Add / remove accounts

```bash
resonix channels add --channel telegram --token <bot-token>
resonix channels remove --channel telegram --delete
```

Tip: `resonix channels add --help` shows per-channel flags (token, app token, signal-cli paths, etc).

## Login / logout (interactive)

```bash
resonix channels login --channel whatsapp
resonix channels logout --channel whatsapp
```

## Capabilities probe

Fetch provider capability hints (intents/scopes where available) plus static feature support:

```bash
resonix channels capabilities
resonix channels capabilities --channel discord --target channel:123
```

Notes:

- `--channel` is optional; omit it to list every channel (including extensions).
- `--target` accepts `channel:<id>` or a raw numeric channel id and only applies to Discord.
- Probes are provider-specific: Discord intents + optional channel permissions; Slack bot + user scopes; Telegram bot flags + webhook; Signal daemon version; MS Teams app token + Graph roles/scopes (annotated where known). Channels without probes report `Probe: unavailable`.

## Resolve names to IDs

Resolve channel/user names to IDs using the provider directory:

```bash
resonix channels resolve --channel slack "#general" "@jane"
resonix channels resolve --channel discord "My Server/#support" "@someone"
resonix channels resolve --channel matrix "Project Room"
```

Notes:

- Use `--kind user|group|auto` to force the target type.
- Resolution prefers active matches when multiple entries share the same name.
